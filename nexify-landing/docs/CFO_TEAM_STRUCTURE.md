## 二、各下属职责定义

### 2.1 财务分析师（nexify-fin-analyst）

**核心职责**：数据采集、成本建模、预算编制，为 CFO 决策提供量化支撑。

**职责范围：**
- 建模 1（AI 安全模型）的成本核算与盈利预测
- 月度/季度财务快报编制
- 现金流预测与资金使用效率分析
- 竞争对手财务数据对标（开源情报）
- 预算执行偏差分析（ variance analysis）
- 定价策略数据支撑

**典型交付物：**
- `MODEL_1_COST_MODEL.xlsx`（成本模型表）
- `BUDGET_VARIANCE_MONTHLY.md`（月度预算偏差报告）
- `PRICING_SENSITIVITY.xlsx`（定价敏感性分析）
- `CASHFLOW_FORECAST.md`（现金流预测，滚动 12 个月）

---

### 2.2 法务顾问（nexify-fin-legal）

**核心职责**：合同审查、知识产权管理、法律风险评估，守护公司法律边界。

**职责范围：**
- 服务协议（TOS）、数据处理协议（DPA）等标准合同起草/审查
- 开源许可证审计（确保模型不侵犯第三方版权）
- 客户企业合同中的财务相关条款审查（赔偿上限、付款条件）
- 知识产权布局建议（专利、软件著作权）
- 合作/投资协议法律风险评估
- 纠纷处理预案（仲裁、诉讼）

**典型交付物：**
- `TOS_DRAFT_v1.md`（服务条款草案）
- `DPA_TEMPLATE.md`（数据处理协议模板）
- `LICENSE_AUDIT_REPORT.md`（开源许可证审计报告）
- `LEGAL_RISK_MATRIX.md`（法律风险矩阵）
- `CONTRACT_REVIEW_SUMMARY.md`（合同审查摘要）

---

### 2.3 合规专员（nexify-fin-compliance）

**核心职责**：监管合规、内控建设、审计对接，确保公司运营符合法律监管要求。

**职责范围：**
- 欧盟 AI 法案（EU AI Act）合规路线图制定与跟踪
- 中国生成式 AI 管理暂行办法合规自查
- 数据跨境传输合规方案（GDPR、CCPA、中国数据安全法）
- ISO 42001 / ISO 27001 认证推进
- 财务内控制度建设（报销、审批权限）
- 对接外部审计机构，准备审计材料
- 合规风险事件记录与整改跟踪

**典型交付物：**
- `EU_AI_ACT_COMPLIANCE_CHECKLIST.md`（欧盟 AI 法案合规清单）
- `INTERNAL_CONTROL_SOP.md`（内控 SOP）
- `COMPLIANCE_RISK_REPORT_QUARTERLY.md`（季度合规风险报告）
- `AUDIT_PREP_PACKAGE.md`（审计材料包）
- `DATA_RETENTION_POLICY.md`（数据留存政策）

---

### 2.4 投资者关系（nexify-fin-ir）

**核心职责**：融资材料准备、投资者沟通、估值管理，支撑 CFO 完成资金筹集。

**职责范围：**
- 商业计划书（BP）财务章节编写与更新
- 估值模型构建（DCF、可比公司法）
- 投资人会议材料准备（Pitch Deck 财务部分）
- 财务数据room 管理（due diligence 数据包）
- 投资人问询应答（财务相关问题）
- 股东关系维护（定期财务更新邮件）
- 融资节奏建议（时机、估值、条款）

**典型交付物：**
- `CAP_TABLE_latest.xlsx`（股权结构表）
- `VALUATION_MODEL.xlsx`（估值模型）
- `IR_PITCH_DECK_FINANCIAL_SECTION.md`（Pitch Deck 财务页）
- `DD_PACKAGE_CHECKLIST.md`（尽职调查材料清单）
- `INVESTOR_UPDATE_TEMPLATE.md`（投资人更新邮件模板）

## 三、启动触发条件

### 3.1 财务分析师启动条件

**自动触发（任务导向）：**
- CFO 要求进行财务建模或数据分析
- CEO 需要成本/收入预测数字
- 需要编制预算、审查报销或分析 Variance
- 定价策略讨论需要数据支撑
- Model 1 商业计划更新需要新数字

**典型任务清单：**
```
□ 建立/更新 Model 1 成本模型
□ 编制季度财务快报
□ 分析 API 定价敏感性（价格下调 20% 对 MRR 影响）
□ 审查本月报销单据合规性
□ 更新现金流预测（12 个月滚动）
□ 对标竞品最新融资数据（Crunchbase 监测）
```

---

### 3.2 法务顾问启动条件

**自动触发（事件导向）：**
- 准备与客户签署服务协议（SaaS/定制项目）
- 开源模型发布前许可证审查
- 融资或股权相关法律文件审查
- 发现潜在合同纠纷或法律风险
- 与合作伙伴签署合作协议
- 员工合同/保密协议（NDA）审查

**典型任务清单：**
```
□ 起草/审查企业客户标准服务协议
□ Model 1 开源发布前许可证审计（确认无 GPL 污染风险）
□ 审查 Term Sheet 中的财务相关条款（清算优先权、反稀释）
□ 起草数据处理协议（DPA）模板
□ 评估客户合同赔偿上限是否合理
□ 准备标准免责声明文本
```

---

### 3.3 合规专员启动条件

**自动触发（时间/事件导向）：**
- 欧盟 AI 法案新进展（需要评估对公司影响）
- 中国监管政策更新（算法推荐、生成式 AI）
- 年度审计启动（准备材料、对接审计师）
- 内控检查（报销违规、审批越权）
- 数据泄露或安全事件发生
- 融资 DD（合规尽调）启动

**典型任务清单：**
```
□ 梳理 EU AI Act 高风险 AI 系统合规现状
□ 准备 ISO 42001 认证自查清单
□ 建立 API 调用数据留存机制（当前保留 30 天）
□ 审查 Model 1 的训练数据来源合规性
□ 制定 GDPR 数据主体权利响应 SOP
□ 季度合规风险报告提交
```

---

### 3.4 投资者关系启动条件

**自动触发（融资导向）：**
- CEO 宣布启动新一轮融资
- 投资人主动接触，表达兴趣
- 财务数据达到融资里程碑（MRR 突破 ¥5万）
- 股东要求定期更新
- 准备参加路演/Demo Day

**典型任务清单：**
```
□ 更新 BP 财务章节（基于最新财务数据）
□ 构建/更新 DCF 估值模型
□ 准备 Pitch Deck 财务页（6张关键数据图表）
□ 整理 Due Diligence 材料包（财务、法律、合规）
□ 撰写上一次投资人更新邮件
□ 监测可比公司最新估值变动
```

> ✅ 主导  ○ 支持  — 不参与

---

### 4.2 成本核算谁来做

**财务分析师**负责：
- GPU 训练成本（云账单汇总 × 实际用量）
- 人力成本分摊（按项目工时）
- 数据采购成本归集
- 运营成本（API 推理、带宽、存储）

**法务顾问**负责：
- 数据采购合同条款（价格条款、授权范围）审查，确保成本合法合规
- 开源数据集许可证审查（避免隐性法律成本）

**合规专员**负责：
- 认证费用预算（ISO 42001、第三方安全审计）
- 合规团队人力成本归集

---

### 4.3 合规审查谁来做

**合规专员**负责：
- EU AI Act 合规差距分析（Govern/MAP/Measure/Manage）
- 中国生成式 AI 管理暂行办法自查
- 数据跨境传输合规方案
- 内控制度（财务审批权限）

**法务顾问**负责：
- 法律文本层面的合规（合同条款、免责声明）
- 监管政策变化的法律影响评估

**财务分析师**负责：
- 合规成本量化（认证费、咨询费纳入预算）

---

### 4.4 定价策略谁来做

**财务分析师**主导：
- 可变成本定价法（token 成本 + 毛利率目标）
- 竞品定价对标（API 价格、功能对比）
- 敏感性分析（价格弹性测试）
- 量级折扣模型设计

**法务顾问**支持：
- 定价方案法律边界（是否涉及掠夺性定价？）
- 退款/赔偿条款设计

**投资者关系**支持：
- 定价故事包装（投资人视角的定价策略说明）
- 市场定位叙事（为什么定这个价格？）

## 五、财务流程协作

### 5.1 年度预算审批流程

```
CEO 战略目标输入
        ↓
财务分析师：编制预算草案（收入、成本、现金流）
        ↓
CFO 初审 → 返回修改（循环 1-2 次）
        ↓
合规专员：审查预算中合规成本是否足够（ISO 认证、审计费）
        ↓
法务顾问：审查预算中法务成本是否合理（外部律师费、合同审查量）
        ↓
CFO 终审 → 提交 CEO
        ↓
CEO 批准 → 进入执行
```

**预算执行监控（每月）：**
- 财务分析师生成 Variance Report
- CFO 判断偏差原因
- 偏差 > 10% → 启动预算调整流程

---

### 5.2 报销审批流程

```
员工提交报销单
        ↓
合规专员：初审（票据合规性、报销标准）
        ↓
通过 → 财务分析师复核（金额准确性）
        ↓
金额 ≤ ¥5,000 → CFO 审批
金额 > ¥5,000 → CEO + CFO 双签
        ↓
出纳付款（每月固定付款日）
        ↓
合规专员：月末报销合规抽查（10% 抽样）
```

**合规红线：**
- 单笔 > ¥10,000 无票报销 → 拒绝
- 疑似关联交易 → 自动升级 CFO + 法务顾问联合审查

---

### 5.3 Model 1 审计配合流程

```
年度审计启动通知
        ↓
合规专员：协调内部，准备审计清单（3 周前）
        ↓
财务分析师：提供财务报表、账务记录、成本分摊明细
        ↓
法务顾问：提供合同台账、诉讼/纠纷清单
        ↓
合规专员：整理合规文档包（EU AI Act、数据安全、许可证）
        ↓
CFO 协调会：预审材料（2 周前）
        ↓
审计师现场审计
        ↓
合规专员：跟进整改项（30 天内完成）
        ↓
CFO 最终审阅审计报告 → CEO 汇报
```

---

### 5.4 融资财务流程

```
融资启动决策（CEO → CFO）
        ↓
财务分析师：更新财务模型（3-way forecast、DCF）
投资者关系：收集/更新融资材料
        ↓
CFO 整合 → 财务数据包（Data Room）
        ↓
法务顾问：审查 Term Sheet 财务条款
合规专员：准备合规 DD 材料
        ↓
路演材料审阅（CFO → CEO）
        ↓
投资人对接
        ↓
法务顾问：SPA/股东协议审查
投资者关系：估值谈判支持
        ↓
CFO 最终条款确认 → CEO 签署
        ↓
交割后：投资者关系 → 股东登记更新（Cap Table）
财务分析师 → 账务处理（股权激励等）
```

## 六、团队运作规范

### 6.1 命名规范

```
nexify-fin-analyst      ← 财务分析师（日常常驻，随需唤醒）
nexify-fin-legal        ← 法务顾问（合同/合规事件触发）
nexify-fin-compliance   ← 合规专员（定期 + 事件触发）
nexify-fin-ir           ← 投资者关系（融资周期触发）
```

### 6.2 生命周期管理

| 场景 | 动作 |
|------|------|
| 常规财务分析 | 启动 analyst → 交付物完成 → 终止 |
| 合同审查 | 启动 legal → 合同签署 → 终止 |
| 季度合规报告 | 启动 compliance → 报告提交 → 终止 |
| 融资周期 | 启动 ir + analyst → 交割完成 → 终止 |
| 年度预算 | 启动 analyst → CEO 批准 → 终止 |

**CFO 原则**：每个下属执行完具体任务后立即终止，不长期保持活跃状态。

### 6.3 文档归档规范

所有下属产出必须归档至 `nexify-landing/docs/`，命名约定：
```
CFO_[角色]_[交付物名称]_[日期].md
示例：
  CFO_ANALYST_BUDGET_VARIANCE_202604.md
  CFO_LEGAL_TOS_DRAFT_v1.md
  CFO_COMPLIANCE_EU_AI_ACT_CHECKLIST.md
  CFO_IR_VALUATION_MODEL.xlsx
```

## 七、团队建设原则

1. **轻量优先**：财务团队以"按需唤醒"模式运作，不养闲人
2. **数据驱动**：所有决策必须有财务模型支撑，杜绝拍脑袋
3. **合规底线**：合规成本是必须成本，不在合规上省钱
4. **价值创造**：财务团队的最终目标是帮公司省钱、赚钱、管好钱
5. **及时收尾**：任务完成后立即终止下属，避免资源浪费
