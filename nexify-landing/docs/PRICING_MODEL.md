## 一、定价哲学

**核心原则：让 AI 民主化 (Democratize AI)**

- **低门槛进入**：免费层足够体验核心价值
- **按需付费**：付费层匹配用户实际需求
- **透明定价**：无隐藏费用，价格清晰
- **价值对齐**：定价反映真实价值交付

## 四、定价心理学应用

### 4.1 锚定效应（Anchoring）

**展示策略：让 Pro 档显得最划算**

```
┌─────────────────────────────────────────────────────────────┐
│                    价格对比（月费）                          │
├─────────────────────────────────────────────────────────────┤
│  Enterprise    $499/月    ████████████████████  (锚点)      │
│                                                             │
│  Pro          $99/月     ████████  ← 节省 $400/月！(80%)    │
│                                                             │
│  Starter       $29/月     ███       (对比后显得"功能不足")   │
│                                                             │
│  Free          $0/月      -         (入门体验)              │
└─────────────────────────────────────────────────────────────┘
```

**话术设计：**
> "Enterprise 的无限调用和定制模型能力适合大型团队。对于大多数专业用户，**Pro 档以 1/5 的价格提供 80% 的核心功能**——这是我们的最佳价值选择。"

**心理学原理：**
- Enterprise 作为高锚点，让 $99 显得"便宜"
- 同时展示节省金额（$400/月），强化价值感
- 避免只展示 Pro 价格（会显得贵）

---

### 4.2 诱饵效应（Decoy Effect）

**设计一个"明显不划算"的中间档作为诱饵：**

| 方案 | 价格 | 调用量 | 单次成本 | 性价比 |
|------|------|--------|---------|--------|
| Starter | $29 | 10,000 次 | $0.0029 | 基准 |
| **Pro（诱饵前）** | **$89** | **30,000 次** | **$0.0030** | ❌ 更贵！ |
| Pro（实际） | $99 | 50,000 次 | $0.0020 | ✅ 最优！ |

**策略：**
- 如果只展示 $89/30K 调用，用户会算出单次成本更高，可能选 Starter
- 实际定价 $99/50K 调用，单次成本更低，引导用户选择 Pro

**对比页面设计：**

```
┌─────────────────────────────────────────────────────────────┐
│  "哪个套餐最适合您？"                                        │
├─────────────────────────────────────────────────────────────┤
│  ○ Free $0/月         - 适合体验和测试                       │
│                                                             │
│  ○ Starter $29/月     - 个人项目和学习                       │
│                        单次调用成本：$0.0029                 │
│                                                             │
│  ● Pro $99/月 ⭐最划算 - 专业应用和商业项目                  │
│     ✓ 调用量是 Starter 的 5 倍                              │
│     ✓ 单次调用成本仅 $0.0020（省 31%）                       │
│     ✓ 高优先级推理，响应更快                                 │
│     【大多数用户的选择】                                     │
│                                                             │
│  ○ Enterprise $499/月 - 企业定制需求                         │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 升级激励机制

**免费 → 付费引导：**

| 触发场景 | 引导策略 |
|---------|---------|
| 调用量达 80% | 提示"您已使用 80% 免费额度，升级获得 10 倍调用" |
| 需要高级模型 | "该模型需要 Starter 或更高套餐，升级仅需 $29/月" |
| 推理排队 > 30s | "升级 Pro 获得高优先级推理，立即响应" |
| 上下文超限 | "您的输入超过 4K tokens，升级 Pro 支持 64K" |

**Starter → Pro 引导：**

| 触发场景 | 引导策略 |
|---------|---------|
| 月调用超量 | "您已超量，升级 Pro 获得 5 倍调用 + 更低单价" |
| 需要批量 | "批量请求功能需要 Pro 或更高套餐" |
| 多次超量购买 | "过去 3 个月您购买了超量包，升级 Pro 可省 $XX" |
| 体验新模型 | "升级 Pro 抢先体验最新 70B 模型" |

**月付 → 年付引导：**

| 策略 | 内容 |
|------|------|
| 首月后提示 | "选择年付立省 $118，相当于免费使用 1.2 个月" |
| 支付页面 | 默认勾选年付选项，展示节省金额 |
| 续费邮件 | "续费年付再享 5% 额外折扣" |

### 4.4 损失厌恶（Loss Aversion）

**话术设计：**

| 场景 | 传统说法 ❌ | 损失厌恶说法 ✅ |
|------|-----------|----------------|
| 升级提示 | "升级 Pro 每月 $99" | "您本月因等待低优先级推理浪费了 **3.5 小时**，升级 Pro 可节省这些时间" |
| 年付推广 | "年付可省 $118" | "不选择年付，您每月多付 $9.8，一年损失 **$118**" |
| 超量提醒 | "您已超量，请购买超量包" | "继续使用当前速度，您的项目可能延迟 **2 天**交付" |
| 续费提醒 | "请续费以继续使用" | "如果现在不续费，您将失去 **已积累的 50,000 条对话历史**" |

## 七、定价风险管理

### 7.1 常见定价风险

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| 定价过高，转化率低 | 中 | 高 | 提供 Founding Member 折扣，快速测试验证 |
| 定价过低，利润薄 | 中 | 高 | 设定最低价格红线，不参与价格战 |
| 竞品价格战 | 低 | 中 | 强调差异化价值，不打价格战 |
| 成本上涨（GPU） | 高 | 高 | 定价预留 20% 缓冲，动态调整超量价格 |
| 免费层滥用 | 中 | 低 | 设置频率限制、CAPTCHA、行为分析 |

### 7.2 价格调整预案

| 场景 | 调整策略 |
|------|---------|
| 成本上涨 > 20% | 调整超量定价，老用户价格锁定 6 个月 |
| 转化率 < 1% | 提供限时折扣，或下调 Pro 档价格 |
| LTV/CAC < 2x | 优化转化漏斗，或提高价格 |
| 竞品降价 | 强调差异化，提供增值服务而非降价 |
