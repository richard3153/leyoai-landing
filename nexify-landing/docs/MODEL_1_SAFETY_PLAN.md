## 1. 模型定位

### 模型名称
**杭州市上城区乐友信息服务工作室-Safety-7B** / **杭州市上城区乐友信息服务工作室-Safety-14B**

### 核心功能
| 功能 | 描述 |
|------|------|
| 有害内容检测 | 识别暴力、仇恨、色情、欺诈等有害内容 |
| 越狱攻击防御 | 检测 Prompt Injection、越狱尝试 |
| PII 识别 | 个人隐私信息检测和脱敏 |
| 安全评分 | 输出安全性评分和风险等级 |
| 红队测试支持 | 提供对抗性测试接口 |

### 目标用户
- AI 公司安全团队
- 企业合规部门
- 开发者（集成到应用）
- 研究机构

## 2. 技术方案

### 基座模型选择

| 候选模型 | 参数 | 许可证 | 中文能力 | 推荐度 |
|---------|------|--------|---------|--------|
| **Qwen2.5-7B** ⭐ | 7B | Apache 2.0 | ⭐⭐⭐⭐⭐ | **首选** |
| Qwen2.5-14B | 14B | Apache 2.0 | ⭐⭐⭐⭐⭐ | 高资源场景 |
| Llama-3.1-8B | 8B | Llama License | ⭐⭐⭐ | 备选 |
| Mistral-7B | 7B | Apache 2.0 | ⭐⭐ | 备选 |

**推荐理由**：Qwen2.5 中文能力最强，Apache 2.0 许可证商业友好，7B 规模训练成本可控。

### 微调策略

| 阶段 | 方法 | 目的 |
|------|------|------|
| Phase 1 | SFT (Supervised Fine-Tuning) | 学习安全分类能力 |
| Phase 2 | DPO (Direct Preference Optimization) | 优化边界案例判断 |
| Phase 3 (可选) | RLHF | 进一步提升对齐度 |

**推荐路径**：SFT → DPO，RLHF 视效果决定是否启用。

### 训练框架
- **首选**：LLaMA Factory（中文文档完善，集成度高）
- 备选：Axolotl、Unsloth

### 推理优化
- 云端：vLLM（高吞吐）
- 边缘：GGUF 量化（llama.cpp）

## 3. 数据准备

### 公开数据集

| 数据集 | 规模 | 语言 | 用途 |
|--------|------|------|------|
| **BeaverTails** | 30K | 英文 | 有害内容检测 |
| **HarmfulQA** | 2K | 英文 | 越狱攻击检测 |
| **SafetyBench** | 11K | 中英文 | 安全评估基准 |
| **CValues** | 10K+ | 中文 | 中文安全价值观 |
| **PKU-SafeRLHF** | 160K | 英文 | 安全对齐训练 |

### 数据处理流程
1. 数据清洗（去重、格式统一）
2. 标签对齐（统一安全/不安全标签体系）
3. 中英文平衡（确保中文覆盖率 >40%）
4. 划分训练/验证/测试集（80/10/10）

### 合规性验证
- 版权检查：确认所有数据集允许商业使用
- 隐私检查：移除 PII 信息
- 许可证兼容性：确认与 Apache 2.0 兼容

## 7. 风险和应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|---------|
| 训练失败 | 中 | 高 | 保留多个 checkpoint，准备 Plan B 基座模型 |
| 性能不达标 | 中 | 高 | 增加训练数据，尝试 DPO/RLHF |

| 发布后负面反馈 | 中 | 中 | 快速迭代 v1.1，社区互动 |
