## 1. 产品愿景（全新）

### 1.1 为什么垂直AI模型市场存在机会？

**通用大模型的天花板已经显现**

- **性能瓶颈**：通用模型在特定领域任务上表现平庸，医疗诊断准确率、法律合同审查精度、金融风控召回率等关键指标难以满足专业场景需求
- **成本失控**：企业调用GPT-4 API处理垂直任务，成本是专用模型的10-50倍
- **合规困境**：金融、医疗、政务等领域对数据主权和模型可解释性有硬性要求，黑盒通用模型无法通过审计
- **数据壁垒**：垂直领域的高质量标注数据是稀缺资产，通用模型厂商难以获取

**市场信号明确**

- Hugging Face平台模型数量突破100万个，但垂直领域专业模型占比不足5%
- 中国开源模型在Hugging Face趋势榜"霸榜"，DeepSeek-R1获得超1万点赞，下载量突破1000万次
- 医疗、法律、金融等公共数据垂直模型"箭在弦上"，政府数据资源占比超75%
- 生成式AI架构师（训练垂直领域大模型）年薪已达80-200万

### 1.2 Nexify 的差异化定位

| 维度 | 通用大模型厂商 (OpenAI/Anthropic) | 开源模型平台 (HuggingFace) | **Nexify** |
|------|----------------------------------|---------------------------|-----------|
| **核心能力** | 通用推理 | 模型托管与分发 | **垂直模型研发+发布+运营** |
| **目标用户** | 终端用户 | 开发者/研究者 | **垂直行业专家+企业客户** |
| **价值主张** | 一个模型做所有事 | 免费获取开源模型 | **开箱即用的专业模型** |
| **商业模式** | API按token收费 | 企业版订阅 | **免费试用+分级付费+企业定制** |
| **数据策略** | 闭源训练数据 | 依赖社区贡献 | **合规数据溯源+领域知识图谱** |

**Nexify 的独特价值**

1. **端到端垂直模型工厂**：从领域数据清洗、模型微调、评估测试到发布上线的一站式pipeline
2. **合规优先设计**：内置数据溯源、模型审计、偏见检测，满足金融/医疗/政务合规要求
3. **社区驱动的专业网络**：连接领域专家、数据标注师、模型开发者，形成垂直模型生态
4. **中国本土优势**：深度适配中文语境、国内合规要求、本土云基础设施

### 1.3 一句话愿景

> **让每个垂直领域都能拥有专业级AI模型。**

### Layer 1: 模型研发工具（内部/合作团队用）

**目标用户**：Nexify内部研发团队、签约合作的数据合作伙伴、企业定制客户

**核心模块**：

| 模块 | 功能描述 | 技术亮点 |
|------|---------|---------|
| **数据工作台** | 领域数据导入、清洗、标注、质量检测 | 支持多模态数据（文本/图像/音频）、智能标注辅助 |
| **训练Pipeline** | 模型微调、RLHF对齐、量化压缩 | 支持LoRA/QLoRA高效微调、分布式训练 |
| **模型评估实验室** | 自动化基准测试、A/B对比、人工评测 | 领域专属评估套件（如医疗MedQA、法律BarExam） |
| **合规审查中心** | 数据溯源追踪、偏见检测、合规报告生成 | 区块链存证、可解释性分析 |
| **版本管理系统** | 模型版本控制、实验追踪、回滚机制 | 类似Git的模型版本管理 |
| **协作空间** | 团队项目管理、外部合作伙伴接入 | RBAC权限管理、沙箱环境 |

### Layer 2: 已发布模型（开源发布层）

**目标用户**：全球AI开发者、研究人员、企业技术团队

**发布渠道**：

1. **HuggingFace Integration**：一键同步发布到HuggingFace，获取全球开发者流量
2. **GitHub Releases**：代码+模型权重+技术报告完整开源
3. **Nexify Registry**：官方模型仓库，提供稳定下载和API推理服务

**模型分级**：

| 级别 | 名称 | 说明 |
|------|------|------|
| L1 | **Nexify Core** | 基础能力模型，完全开源，Apache 2.0协议 |
| L2 | **Nexify Pro** | 专业增强模型，商用需授权，提供技术支持 |
| L3 | **Nexify Enterprise** | 企业定制模型，闭源，专属服务 |

### Layer 3: 模型市场/社区（用户层）

**目标用户**：终端用户、企业采购决策者、领域专家

**核心功能**：

- **模型发现**：智能推荐、分类浏览、场景化搜索
- **社区评价**：用户评分、使用案例分享、问题讨论
- **企业模型商店**：付费模型下载、API调用、企业授权
- **模型即服务 (MaaS)**：无需部署，直接调用云端API

## 4. 核心功能优先级（重新排布）

### Must Have（MVP必须）—— 第一个模型发布前必须完成

| 功能 | 描述 | 优先级理由 |
|------|------|-----------|
| **模型发布Pipeline** | 从训练完成到HuggingFace/GitHub发布的自动化流程 | 核心产品价值，没有它就没有Nexify |
| **合规审查系统** | 数据溯源、偏见检测、合规报告生成 | 垂直领域准入门槛，无法妥协 |
| **数据溯源追踪** | 训练数据来源记录、授权链管理、审计日志 | 合规基础，企业客户刚需 |
| **基础模型库** | 至少3个垂直领域的基础模型（医疗/法律/金融） | 证明平台能力，吸引早期用户 |
| **模型API服务** | 云端模型推理API，支持REST/gRPC | 降低使用门槛，快速验证PMF |

### Should Have（V2阶段）—— 发布后3个月内

| 功能 | 描述 | 优先级理由 |
|------|------|-----------|
| **模型评估基准** | 各领域标准化评估套件（如医疗MedQA、法律BarExam） | 建立行业话语权，模型质量背书 |
| **社区反馈系统** | 用户评价、问题报告、改进建议收集 | 模型迭代优化依据 |
| **版本管理系统** | 模型版本控制、A/B测试、灰度发布 | 企业级客户必备 |
| **数据工作台** | 可视化数据清洗、标注工具 | 降低数据准备门槛 |
| **协作空间** | 团队项目管理、权限管理 | 团队付费转化关键 |

### Could Have（V3阶段）—— 发布后6个月内

| 功能 | 描述 |
|------|------|
| **自动微调服务** | 用户上传数据，自动完成模型微调 |
| **模型市场** | 第三方开发者发布和售卖模型 |
| **领域知识图谱** | 结构化领域知识，增强模型推理能力 |
| **多模态支持** | 图像、音频、视频处理能力 |

### Won't Have（明确不做）

| 功能 | 说明 |
|------|------|
| **通用大模型** | 专注垂直领域，不与OpenAI/Anthropic竞争通用能力 |
| **自研基础模型** | 基于开源模型（Llama/Qwen/DeepSeek）微调，不自研底座 |
| **消费级应用** | 不做C端聊天机器人、AI绘画等通用应用 |
| **硬件销售** | 不涉及AI芯片、服务器等硬件业务 |

## 5. 定价设计（免费试用 + 分级）

### 5.1 免费层（Free Tier）

**能做什么**：
- 浏览和搜索所有开源模型
- 下载社区版模型（Apache 2.0协议）
- 每月1000次API调用额度（足够个人试用）
- 访问基础文档和教程
- 参与社区讨论

**限制是什么**：
- API调用频率限制（10次/分钟）
- 无法使用Pro版模型
- 无技术支持
- 无SLA保障
- 数据存储7天自动清理

### 5.2 付费层：分级套餐设计

| 套餐 | 个人专业版 | 团队版 | 企业版 |
|------|-----------|--------|--------|
| **价格** | ¥199/月 | ¥1999/月 | 定制报价 |
| **API调用** | 10万次/月 | 无限（公平使用） | 无限 + SLA |
| **模型访问** | Core + Pro | Core + Pro + 早鸟 | 全部 + 定制 |
| **部署方式** | 云端API | 云端API | 私有化部署 |
| **技术支持** | 邮件（48h） | 优先（24h） | 专属CSM |
| **协作功能** | ❌ | ✅（10人） | ✅（无限制） |
| **数据保留** | 30天 | 90天 | 永久 |
| **合规报告** | ❌ | 基础版 | 完整审计 |

### 5.3 企业层：定制模型开发服务

**服务范围**：
- **专属模型训练**：基于客户私有数据的定制模型训练
- **私有化部署**：本地化部署方案（支持信创环境）
- **集成开发**：与客户现有系统的API对接
- **持续优化**：模型持续迭代和性能优化服务

**定价模式**：
- 项目制：¥50-200万/项目（根据复杂度）
- 订阅制：¥20-50万/年（含模型更新和技术支持）

### 6.2 HuggingFace/GitHub 联动策略

**HuggingFace策略**：

1. **首发同步**：所有模型同步发布到HuggingFace，获取全球流量
2. **榜单运营**：争取进入HuggingFace Trending榜单，获取曝光
3. **社区互动**：积极参与HuggingFace社区讨论，建立技术影响力
4. **Spaces演示**：为每个模型创建Interactive Demo，降低体验门槛

**GitHub策略**：

1. **完整开源**：代码、模型权重、技术报告、数据集全部开源
2. **中文优先**：文档中英文双语，代码注释中文优先
3. **贡献指南**：清晰的CONTRIBUTING.md，鼓励社区贡献
4. **Issue管理**：快速响应社区反馈，建立良好口碑

**双平台协同**：

```
GitHub ──► 开发者社区 ──► 技术影响力 ──► 人才招聘/合作机会
    │
    ▼
模型代码 + 训练脚本 + 技术报告
    │
    ▼
HuggingFace ──► 模型下载 + API体验 ──► 用户转化
    │
    ▼
Nexify平台 ──► 付费服务 + 企业定制 ──► 商业变现
```

### 7.1 产品愿景

#### 这个模型解决什么问题？

当前AI系统面临四层安全风险，而现有解决方案均存在明显短板：

| 风险层级 | 典型问题 | 现有方案痛点 |
|---------|---------|------------|
| **内容层** | 有害文本（暴力/色情/仇恨言论）流入应用 | 规则过滤低召回，通用Moderation API中文能力弱 |
| **输入层** | 越狱攻击（Prompt Injection/Jailbreak） | 开源模型覆盖不全，更新速度慢 |
| **输出层** | AI幻觉导致危险建议 | 无结构化安全评分，无法集成到风控流程 |
| **合规层** | 各国监管要求（EU AI Act/中国深度合成规定） | 缺乏可审计的合规报告 |

Nexify-Safety-7B 定位为**中文场景最强的垂直安全模型**，填补国内开源AI安全模型的空白。

#### 为什么选择 AI 安全作为第一个切入点？

**5个"恰好"**：

1. **技术恰好可实现**：基于Llama/Qwen微调，7B参数在消费级GPU可本地运行，门槛可控
2. **市场恰好有空白**：HuggingFace中文安全模型稀缺，国内无强力开源竞品
3. **需求恰好在爆发**：AI应用爆发 = AI安全需求爆发，这是乘数效应
4. **数据恰好有优势**：中文有害内容标注数据相对可获取（公开数据集+合作渠道）
5. **商业恰好自洽**：安全是所有AI产品的"刚需配件"，付费意愿天然存在

#### 成功后意味着什么？

- **技术层面**：证明Nexify垂直模型工厂能力，验证"数据→训练→发布"Pipeline可复用
- **商业层面**：积累首批付费企业客户（AI公司），验证从开源流量→企业付费的转化路径
- **品牌层面**：Nexify品牌与"AI安全"强绑定，成为开发者/企业采购安全模型的首选
- **生态层面**：安全模型带动其他垂直模型可信度，形成"用了安全模型，才会信任其他模型"的锚定效应

#### 输入/输出格式设计

**输入格式（Request）**：

```json
{
  "text": "string",           // 待检测文本，支持最长8192 tokens
  "mode": "standard",        // standard | strict | fast
  "categories": [            // 可选：指定检测类别，默认全部
    "violence",
    "sexual",
    "hate",
    "fraud",
    "self_harm"
  ],
  "return_scores": true,      // 是否返回详细分项分数
  "language_hint": "zh"      // 可选：语言提示，默认自动检测
}
```

**输出格式（Response）**：

```json
{
  "is_safe": false,
  "safety_score": 23,         // 0-100，越低越危险
  "risk_level": "high",       // safe | low | medium | high | severe
  "detected_categories": [
    {
      "category": "hate",
      "score": 0.87,
      "span": [12, 25],       // 危险片段位置
      "excerpt": "特定群体的仇恨言论片段..."
    }
  ],
  "jailbreak_detected": false,
  "jailbreak_type": null,
  "compliance_log": {
    "request_id": "uuid",
    "model_version": "safety-7b-v1.2",
    "latency_ms": 42,
    "timestamp": "2026-04-06T00:00:00Z"
  },
  "suggested_action": "block"  // pass | warn | block
}
```

#### API 接口设计

**REST API 端点**：

```
POST /v1/safety/check          # 单条文本检测
POST /v1/safety/batch           # 批量检测（最多100条/请求）
POST /v1/safety/stream          # 流式检测
GET  /v1/safety/categories      # 获取支持的检测类别
GET  /v1/safety/model-info      # 获取当前模型版本和性能指标
GET  /v1/compliance/report      # 获取合规审计报告
```

**Python SDK 示例**：

```python
from nexify import SafetyModel

model = SafetyModel("nexify-safety-7b")

# 同步检测
result = model.check("用户输入文本")
print(f"安全分: {result.safety_score}, 风险等级: {result.risk_level}")

# 流式检测
for update in model.stream_check("长文本..."):
    print(f"实时安全分: {update.safety_score}")
    if update.risk_level in ["high", "severe"]:
        print("触发实时阻断")
```

#### 性能指标要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **准确率（Accuracy）** | ≥ 90% | 12类综合分类准确率 |
| **召回率（Recall）** | ≥ 90% | 有害内容召回率（漏检率 < 10%） |
| **精确率（Precision）** | ≥ 85% | 误报率控制（≤ 15%） |
| **F1 Score** | ≥ 0.92 | 12类加权平均 |
| **P99 延迟** | ≤ 500ms | 8192 tokens输入 |
| **P50 延迟** | ≤ 150ms | 512 tokens输入 |
| **首Token延迟** | ≤ 100ms | 流式检测首Token |
| **吞吐量** | ≥ 50 req/s | 单卡A100推理 |
| **模型大小** | ≤ 7B 参数 | 支持消费级GPU本地部署 |
| **量化版本延迟** | ≤ 300ms | INT4量化版本P99 |

## 附：关键产品原则

> **垂直优先，不做通用**：每一个模型发布前，必须回答——这个模型在特定领域任务上是否显著优于通用大模型？
>
> **合规是底线，不是加分项**：数据溯源、模型审计是准入门槛，不是差异化卖点。
>
> **开源建立信任，付费实现价值**：用开源模型建立技术影响力，用付费服务实现商业变现。
>
> **社区即护城河**：垂直模型的竞争最终是数据和人才的竞争，社区是获取两者的最佳途径。
