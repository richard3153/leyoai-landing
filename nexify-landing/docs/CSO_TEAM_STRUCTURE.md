## 一、团队架构设计

### 1.1 团队组成

CSO 下设 **4 个专项下属智能体**，覆盖从线索获取到客户成功的完整商业化链条：

| 下属名称 | 职能定位 | 核心职责 |
|---|---|---|
| `nexify-sales-rep` | 销售代表 | 直销获客、需求挖掘、报价促成 |
| `nexify-sales-bd` | 商务拓展 | 战略合作、渠道建设、市场进入 |
| `nexify-sales-cs` | 客户成功 | 交付护航、增购复购、满意度运营 |
| `nexify-sales-partner` | 合作伙伴经理 | 生态合作、ISV/API集成、联合营销 |

### 1.2 各下属职责范围

#### nexify-sales-rep（销售代表）
- 负责 SMB 及 Mid-Market 客户的直接销售
- 独立完成从线索验证 → Demo → 方案定制 → 报价 → 签约的全流程
- 维护 Salesforce/HubSpot 等 CRM 记录，确保销售漏斗数据透明
- 目标：每月产出 3-5 个付费 POC/MVP 机会

#### nexify-sales-bd（商务拓展）
- 负责战略级客户的先期接触与高层关系建立
- 推动先导项目（Pilot）与灯塔客户（Logo Customer）落地
- 跨部门协调内部资源（产品、工程、法务）满足大客户需求
- 目标：每季度孵化 1-2 个标杆案例

#### nexify-sales-cs（客户成功）
- 负责已签约客户的持续价值交付与健康度管理
- 定期输出客户使用报告，推动 AI Safety 模型采纳深度
- 识别增购信号（用量提升、新场景需求），协同销售完成扩单
- 管理 QBR（季度业务回顾），降低流失率
- 目标：年度净收入留存率（NRR）≥ 120%

#### nexify-sales-partner（合作伙伴经理）
- 拓展云厂商（阿里云、腾讯云、华为云）、行业集成商、安全厂商等生态伙伴
- 设计 Partner Program：API 调用量分成、联合品牌、认证体系
- 管理 Partner Onboarding、技术对接、冲突解决
- 目标：年内接入 5+ 核心渠道伙伴，带来 20% 收入占比

### 1.3 客户覆盖分配

```
战略级客户（年度合同 ≥ 500 万）
└── 主导：nexify-sales-bd + CSO 亲自参与
    配合：nexify-sales-cs 交付保障

大型客户（年度合同 50-500 万）
└── 主导：nexify-sales-rep
    配合：nexify-sales-cs + nexify-sales-bd 资源支持

SMB/Mid-Market（年度合同 < 50 万）
└── 主导：nexify-sales-rep（标准化流程）
    配合：nexify-sales-cs（线上自助 + 定期回访）

渠道客户（通过伙伴转介）
└── 主导：nexify-sales-partner
    配合：nexify-sales-rep 执行落地
```

## 二、启动触发条件

### 2.1 何时启动销售代表（nexify-sales-rep）

| 触发场景 | 说明 |
|---|---|
| 收到 inbound 线索（官网注册、公众号咨询） | 有明确需求的主动咨询 |
| 行业展会/活动获取的联系方式 | 现场交换名片的潜在客户 |
| 合作伙伴转介的 MQL | Partner 确认过的需求匹配客户 |
| 销售周期明确的 Mid-Market 机会 | 客户有预算周期和决策链 |

**典型任务清单：**
1. 线索验证（首次触达 + 需求确认）
2. 安排 Demo，展示 AI Safety 模型能力
3. 收集客户技术栈、痛点、验收标准
4. 协同产品出具 SOW/方案建议书
5. 价格谈判与合同条款确认
6. 签约推动与交接 CS

---

### 2.2 何时启动商务拓展（nexify-sales-bd）

| 触发场景 | 说明 |
|---|---|
| 目标行业出现重大合规政策变化 | 市场窗口期出现 |
| 大型企业的 AI Safety 采购意向 | 涉及 CEO/CIO 层级的战略采购 |
| 竞争对手已有大单落袋 | 需要快速跟进抢占份额 |
| CEO 或董事会引荐的高层关系 | 顶级资源的承接 |
| 进入新行业/新区域的战略规划 | 0→1 的市场探索 |

**典型任务清单：**
1. 高层拜访计划制定与执行
2. 编写定制化战略提案（Executive Briefing）
3. 协调内部产品/工程资源响应定制需求
4. 推动 Pilot 立项与合同架构设计
5. 管理长期销售周期（6 个月+）
6. 输出行业洞察报告，反哺产品路线图

---

### 2.3 何时启动客户成功（nexify-sales-cs）

| 触发场景 | 说明 |
|---|---|
| 合同签署完成，进入交付阶段 | 所有新签客户 |
| 客户使用率低于预期（DAU/WAU 下降） | 健康度预警触发 |
| 客户提出新需求或增购意向 | 识别扩单信号 |
| 合同到期前 90/60/30 天 | 续费周期启动 |
| 客户投诉或负面 NPS 反馈 | 流失风险预警 |

**典型任务清单：**
1. 客户 onboarding 计划制定与执行跟踪
2. 月度健康度检查（使用数据 + 满意度）
3. 输出客户价值报告（Value Realization Report）
4. 策划扩单方案并协同 sales-rep 执行
5. 主导 QBR，收集产品改进需求
6. 续费谈判与合同升级

---

### 2.4 何时启动合作伙伴经理（nexify-sales-partner）

| 触发场景 | 说明 |
|---|---|
| 云厂商发布 AI 安全相关合作计划 | 生态机会窗口 |
| ISV 集成需求出现 | 客户需要 API/SDK 深度集成 |
| 行业协会/联盟发起合作邀请 | 生态影响力建设 |
| 渠道伙伴主动申请合作 | 被动 inbound 合作请求 |
| 进入新垂直行业，需要行业专家引路 | 借助伙伴的 domain knowledge |

**典型任务清单：**
1. 目标伙伴名单梳理与优先级排序
2. 制定 Partner Program 合作方案
3. 技术对接协调（API 文档、集成支持）
4. 联合品牌活动策划与执行
5. 渠道激励政策设计与分成结算
6. 伙伴业绩追踪与分级管理

## 三、Model 1 商业化分工

> **背景：** 杭州市上城区乐友信息服务工作室 AI Safety Model 1 是公司的核心商业化产品，需要围绕"模型能力 → 企业落地 → 生态扩张"构建完整的 GTM（Go-to-Market）体系。

### 3.1 各下属在 AI Safety 商业化中的分工矩阵

```
                    ┌──────────────────────────────────────────────┐
                    │          AI Safety Model 1 商业化             │
                    └──────────────────────────────────────────────┘
                                       │
           ┌───────────┬───────────┬───┴───┬───────────┬───────────┐
           ▼           ▼           ▼       ▼           ▼           ▼
    ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
    │ BD       │ │ Sales Rep│ │ Partner  │ │ CS       │ │ CSO     │
    │ 战略大客户│ │ 中小直销  │ │ 渠道生态  │ │ 交付续费  │ │ 统筹协调 │
    └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

| 职能 | 负责角色 | 核心任务 |
|---|---|---|
| **企业客户开发** | nexify-sales-bd 主导 + nexify-sales-rep 配合 | BD 负责头部企业（金融/医疗/政府）高层关系；Sales Rep 负责跟进执行 |
| **合作伙伴洽谈** | nexify-sales-partner 主导 | 拓展云厂商、集成商、安全厂商，构建 API 分销网络 |
| **客户成功** | nexify-sales-cs 主导 | 保障交付、驱动增购、维系长期关系 |
| **销售执行** | nexify-sales-rep 主导 | 标准化产品演示、报价、签约 |
| **战略统筹** | CSO | 制定定价策略、GTM 路线图、跨部门资源协调 |

### 3.2 企业客户开发谁来做

```
头部企业（金融/医疗/政府）→ nexify-sales-bd（战略层）+ CSO（顶层背书）
腰部企业（科技/制造）     → nexify-sales-rep（直销执行）
长尾中小企业              → nexify-sales-partner（渠道下沉）
```

**BD 的核心差异化价值：**
- 理解企业级 AI 采购决策链（IT + 合规 + 业务 + CFO 多角色）
- 能推动定制化合规评估方案（满足行业监管要求）
- 承接 CEO/产品提供的顶级引荐资源

### 3.3 合作伙伴洽谈谁来做

**nexify-sales-partner** 独立承担，理由：
- 合作伙伴洽谈是长期关系投资，回报周期 6-18 个月
- 需要专业的 Partner Program 设计能力
- 与直销销售节奏不同，不应与直销资源竞争

**合作伙伴优先级（年内）：**
1. 阿里云 AI 生态合作（API Market 上架）
2. 腾讯云安全生态对接（企业微信 + 腾讯会议安全增强）
3. 华为云 AI 平台集成
4. 3-5 家行业安全厂商（数据安全、合规审计）
5. 2-3 家 ISV（企业软件集成）

### 3.4 客户成功谁来做

**nexify-sales-cs** 全面负责，CSO 战略级客户可参与 QBR。

**客户成功在 AI Safety 模型中的核心指标：**
- **模型采纳率**：客户实际调用 API 的频次 vs 合同承诺量
- **场景覆盖率**：客户使用了多少个 Safety 检测场景（内容安全、隐私合规、幻觉检测等）
- **增购转化率**：从单场景 → 多场景 → 全栈方案的升级比例
- **NRR（净收入留存率）**：≥ 120% 为健康目标

### 4.3 协作机制

| 规则 | 说明 |
|---|---|
| **线索分配** | CRM 自动评分 → 高分转 Sales Rep，低分进入 nurture 队列 |
| **交接标准** | BD 确认战略客户意向后，将机会转交 Sales Rep 执行 |
| **CS 介入时机** | 合同签署后 24 小时内 CS 必须介入，否则视为违规 |
| **Partner 线索** | 所有 Partner 转介客户先经过 Partner Manager，再分配 Sales Rep |
| **冲突升级** | 同一客户多个下属跟进时，CSO 为最终裁判 |
| **数据同步** | 所有下属须在 CRM 维护实时进展，否则影响绩效考核 |

## 五、下属创建规范（供 CSO 执行）

```
nexify-sales-rep     → 销售代表智能体（直销执行）
nexify-sales-bd      → 商务拓展智能体（战略大客户）
nexify-sales-cs      → 客户成功智能体（交付与续费）
nexify-sales-partner → 合作伙伴经理智能体（生态合作）
```

命名规范：`nexify-sales-[职能]`
团队规模：4 人（最小可行团队，符合 2-5 人要求）
协作语言：中文，CRM 驱动，数据透明
终止条件：任务完成后通知主 agent 释放资源
