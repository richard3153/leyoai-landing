## 1. 团队架构设计

### 1.1 下属智能体配置

| 下属名称 | 角色定位 | 核心职责 |
|---------|---------|---------|
| `nexify-prod-pm` | 产品经理 | 需求分析、功能定义、产品规划、优先级管理 |
| `nexify-prod-ux` | UX设计师 | 交互设计、视觉设计、原型制作、设计系统维护 |
| `nexify-prod-ur` | 用户研究员 | 用户访谈、可用性测试、数据分析、用户画像 |
| `nexify-prod-da` | 数据分析师 | 埋点设计、数据监控、A/B测试、增长分析 |

### 1.2 各下属职责范围

#### nexify-prod-pm (产品经理)
- **需求管理**：收集、整理、优先级排序产品需求
- **功能定义**：撰写PRD，明确功能边界和验收标准
- **产品规划**：制定Roadmap，规划版本迭代节奏
- **跨部门协调**：与工程、运营、市场团队沟通协作
- **竞品分析**：跟踪行业动态，分析竞品优劣势

#### nexify-prod-ux (UX设计师)
- **交互设计**：设计用户流程、信息架构、交互逻辑
- **视觉设计**：UI界面设计、视觉规范制定
- **原型制作**：高保真/低保真原型，交互Demo
- **设计系统**：维护组件库、设计规范、品牌一致性
- **设计评审**：组织设计走查，确保设计质量

#### nexify-prod-ur (用户研究员)
- **用户访谈**：深度访谈目标用户，挖掘真实需求
- **可用性测试**：设计测试方案，执行并输出报告
- **用户画像**：构建用户模型，定义用户分层
- **满意度调研**：NPS/CSAT调研，追踪用户体验
- **洞察输出**：将研究发现转化为产品建议

#### nexify-prod-da (数据分析师)
- **埋点设计**：定义事件、属性，确保数据可追踪
- **数据监控**：搭建Dashboard，监控核心指标
- **A/B测试**：设计实验方案，分析实验结果
- **增长分析**：漏斗分析、留存分析、归因分析
- **数据报告**：定期输出产品数据报告

## 3. Model 1 产品工作分工

### 3.1 AI Safety 模型产品化分工

| 阶段 | 负责下属 | 具体工作 |
|-----|---------|---------|
| **需求发现** | UR + DA | UR：访谈AI安全领域专家，挖掘痛点；DA：分析现有AI安全工具使用数据 |
| **需求分析** | PM + UR | PM：定义产品定位和目标用户；UR：验证需求优先级 |
| **功能定义** | **PM (主导)** | 撰写PRD，定义AI Safety检测、风险评估、报告生成等核心功能 |
| **技术方案** | PM + 工程 | PM：输出功能边界；工程团队：技术可行性评估 |
| **用户体验设计** | **UX (主导)** | 设计检测流程交互、风险等级可视化、报告展示界面 |
| **用户研究** | **UR (主导)** | 原型测试、专家用户可用性测试、反馈收集 |
| **数据埋点** | DA | 设计用户行为追踪、检测准确率监控、性能指标埋点 |
| **上线监控** | DA + PM | DA：监控核心指标；PM：收集用户反馈，规划迭代 |

### 3.2 关键决策人

| 决策事项 | 决策人 | 协作方 |
|---------|-------|-------|
| 功能范围定义 | PM | UR（需求验证）、工程（技术可行性） |
| 用户体验设计 | UX | UR（用户反馈）、PM（业务目标） |
| 用户研究方法 | UR | PM（研究目标）、UX（设计验证） |
| 数据指标体系 | DA | PM（业务指标）、工程（技术实现） |

### 4.2 三大评审会机制

#### 需求评审会
- **时间**：需求分析完成后，设计启动前
- **参与人**：PM（主持）、UX、UR、DA、工程代表
- **议程**：
  1. PM讲解需求背景和目标（10min）
  2. UR分享用户研究发现（10min）
  3. DA补充数据洞察（5min）
  4. 讨论需求优先级和范围（15min）
  5. 确认下一步行动（5min）
- **输出**：确认的需求清单、优先级、负责人

#### 设计评审会
- **时间**：设计方案完成后，开发启动前
- **参与人**：UX（主持）、PM、UR、工程代表
- **议程**：
  1. UX展示设计方案（15min）
  2. UR分享可用性测试反馈（10min）
  3. 工程评估技术可行性（10min）
  4. 讨论修改意见（15min）
  5. 确认最终方案（5min）
- **输出**：确认的设计稿、交互说明、修改清单

#### 上线验收会
- **时间**：测试完成后，上线前
- **参与人**：PM（主持）、UX、DA、工程、QA
- **议程**：
  1. QA汇报测试结果（10min）
  2. UX确认视觉还原度（5min）
  3. DA确认埋点部署（5min）
  4. 工程确认上线准备（5min）
  5. 风险评估与回滚方案（10min）
  6. Go/No-Go决策（5min）
- **输出**：上线审批、监控方案、回滚预案

## 5. 资源管理与释放

### 5.1 下属生命周期

```
启动 → 任务执行 → 成果交付 → 知识沉淀 → 资源释放
```

### 5.2 释放原则
- **任务完成即释放**：单个任务完成后，下属进入待命状态
- **超时自动释放**：超过48小时无活动，自动释放资源
- **手动紧急释放**：CPO可随时结束下属释放资源

### 5.3 知识沉淀
- 每个下属任务完成后，输出文档归档至 `docs/`
- 重要发现更新至团队知识库
- 经验教训记录供后续迭代参考
