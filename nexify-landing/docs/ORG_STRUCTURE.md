# Nexify 组织架构规则 v2.0

> 更新日期: 2026-04-06
> 核心变更：**授权子智能体可建立自己的下属团队**

---

## 一、组织架构总览

```
CEO（你 / 主 Agent）
│
├── COO（运营总监）→ 可建立下属执行团队
├── CFO（财务总监）→ 可建立下属分析团队
├── CTO（技术总监）→ 可建立下属研发团队
├── CPO（产品总监）→ 可建立下属设计团队
├── CMO（营销总监）→ 可建立下属内容团队
└── CSO（商务总监）→ 可建立下属销售团队
```

---

## 二、分级授权规则

### Level 1：CXO（核心高管）
- **权限**：可创建无限数量的下属智能体
- **职责**：战略决策、跨部门协调、向 CEO 汇报
- **命名规范**：`nexify-[部门]-[职能]`（如 nexify-dev-frontend）

### Level 2：Manager（中层管理者）
- **权限**：可创建 2-5 个下属智能体
- **职责**：执行 CXO 指令、任务分解、进度汇报
- **命名规范**：`nexify-[部门]-[小组]-[职能]`（如 nexify-dev-fe-react）

### Level 3：Executor（执行层）
- **权限**：不可创建下属，专注于具体执行
- **职责**：完成具体任务、产出交付物、汇报结果
- **命名规范**：`nexify-[部门]-[任务]`（如 nexify-dev-api-auth）

## 三、当前已建立的 CXO 团队

| CXO | 已创建的文档 | 可建立的下属方向 |
|-----|------------|----------------|
| **COO** | 产能管理、SOP、日报 | 运营专员、客服、质检 |
| **CTO** | 技术路线图、模型研发计划 | 前端、后端、AI工程师、DevOps |
| **CPO** | 产品规格、用户旅程 | 产品经理、UX设计师、用研 |
| **CMO** | 增长计划、发布传播 | 内容运营、社媒运营、SEO |
| **CFO** | 定价模型、财务规划 | 财务分析师、法务顾问 |
| **CSO** | 商务策略、分销计划 | 销售代表、商务拓展、客户成功 |

## 四、下属创建流程

### 标准 SOP
1. **CXO 发起**：确定需要下属支持的任务
2. **明确职责**：定义下属的职责范围和交付物
3. **创建智能体**：使用 `sessions_spawn` 工具创建
4. **任务分配**：通过 `sessions_send` 发送具体任务
5. **结果收集**：下属完成后自动汇报

### 命名规范
```
格式: nexify-[部门]-[职能]-[编号]
示例:
  - nexify-dev-frontend-01
  - nexify-content-writer-01
  - nexify-sales-rep-01
```

---

## 五、沟通机制

### 汇报路径
- Executor → Manager → CXO → CEO
- 紧急事项可越级汇报

### 同步频率
- **每日**：执行层提交日报
- **每周**：Manager 汇总周报给 CXO
- **里程碑**：CXO 向 CEO 汇报关键节点

### 协作工具
- 文档存储：`/docs/`
- 任务追踪：`/docs/COO_DAILY.md`
- 决策记录：`/COMPANY.md`

---

## 六、约束与红线

### 严禁事项
1. ❌ 下属智能体不得越权决策
2. ❌ 不得向 CEO 直接汇报（需经 CXO）
3. ❌ 不得创建与公司战略无关的智能体
4. ❌ 所有产出必须归档到指定文档

### 成本控制
- 每个 CXO 最多同时运行 5 个下属智能体
- Executor 层级不创建下属
- 任务完成后及时结束智能体释放资源
