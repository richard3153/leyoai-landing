## 详细用户旅程

### Stage 1: 发现阶段（Awareness）

**用户状态**：还不知道Nexify，有垂直AI模型需求但不知道去哪里找

| 触点 | 用户行为 | 用户想法 | 产品机会 |
|------|---------|---------|---------|
| **HuggingFace浏览** | 搜索医疗/法律/金融相关模型 | "有没有专门针对中文医疗场景的模型？" | SEO优化，确保Nexify模型在搜索结果前列 |
| **GitHub探索** | 浏览Awesome-LLM列表 | "这些通用模型在我的场景效果不好" | 在相关repo中提及Nexify垂直模型 |
| **技术社区** | 在知乎/CSDN搜索垂直AI应用 | "怎么把AI应用到我的专业领域？" | 发布垂直领域AI应用案例文章 |
| **同行推荐** | 听同事/朋友提及 | "听说有个专门做垂直模型的平台" | 设计推荐奖励机制 |
| **行业会议** | 参加医疗AI/法律科技峰会 | "有没有即拿即用的专业模型？" | 积极参与行业会议，建立品牌认知 |

**关键指标**：
- 品牌搜索量
- HuggingFace模型页访问量
- GitHub repo star数

---

### Stage 2: 认知阶段（Interest）

**用户状态**：知道Nexify，正在了解产品能力和价值

| 触点 | 用户行为 | 用户想法 | 产品机会 |
|------|---------|---------|---------|
| **落地页访问** | 浏览产品介绍 | "这个平台能做什么？" | 清晰的价值主张，3秒传达核心信息 |
| **模型卡片查看** | 查看模型详情页 | "这个模型在我的场景表现如何？" | 提供详细的Benchmark数据和案例 |
| **技术博客阅读** | 阅读模型发布文章 | "他们是怎么训练这个模型的？" | 透明化训练过程，建立技术信任 |
| **Demo体验** | 试用在线Demo | "实际效果怎么样？" | 提供零门槛的在线体验 |
| **文档浏览** | 查看API文档 | "接入难度如何？" | 提供清晰的Quick Start指南 |

**用户决策关键问题**：
1. 这个模型比通用模型好在哪里？
2. 使用成本高吗？
3. 数据安全吗？
4. 技术支持怎么样？

**产品应对**：
- 首页首屏：一句话价值主张 + 3个核心优势
- 模型详情页：Benchmark对比 + 真实案例 + 用户评价
- 定价页：透明的免费额度 + 清晰的升级路径
- 安全页：合规认证 + 数据处理方式说明

---

### Stage 3: 试用阶段（Trial）

**用户状态**：决定尝试，完成注册并开始使用

#### Step 3.1: 注册账号（目标：降低注册门槛）

| 用户行为 | 可能的 friction | 产品优化 |
|---------|----------------|---------|
| 点击"开始使用" | 需要填写太多信息 | 支持GitHub/Google一键登录 |
| 填写注册表单 | 邮箱验证流程繁琐 | 可选邮箱验证，先体验后完善 |
| 阅读服务条款 | 担心数据隐私 | 明确的数据使用政策，突出"用户数据不用于训练" |

#### Step 3.2: 首次体验（目标：Aha Moment在5分钟内）

**理想路径**：

```
注册成功
    │
    ▼
欢迎页 ──► 选择领域（医疗/法律/金融/其他）
    │
    ▼
推荐模型 ──► 选择感兴趣的场景
    │
    ▼
一键试用 ──► 在线Demo / API Playground
    │
    ▼
获得结果 ──► Aha Moment!
    │
    ▼
引导下一步 ──► 下载模型 / 查看文档 / 加入社区
```

**Aha Moment定义**：
- 医疗用户：输入症状描述，获得准确的诊断建议
- 法律用户：上传合同，获得关键条款提取和风险提示
- 金融用户：输入公司名，获得自动生成的研报框架

**防流失设计**：
- 如果用户5分钟内没有完成首次调用，触发帮助弹窗
- 提供"一键示例"功能，无需输入即可体验
- 实时客服入口，解答使用问题

#### Step 3.3: 完成首次任务（目标：建立使用习惯）

| 用户行为 | 产品支持 | 成功信号 |
|---------|---------|---------|
| 下载模型 | 提供多格式下载（PyTorch/Safetensors/GGUF） | 下载完成率 > 80% |
| API调用 | 提供多语言SDK（Python/Node/Java） | 首次API调用成功率 > 90% |
| 查看结果 | 结果可视化，支持导出 | 结果满意度 > 4.0/5 |
| 查阅文档 | 文档内搜索、相关推荐 | 文档停留时间 > 2分钟 |

## 3个关键转化节点及对应的产品动作

### 转化节点1：注册 → 首次成功调用（Trial Conversion）

**当前行业基准**：SaaS产品平均20-30%，优秀产品可达40%+

**Nexify目标**：≥ 40%

**流失风险**：
- 注册后没有立即找到感兴趣的模型
- Demo体验效果不佳
- API调用遇到技术问题

**产品动作**：

| 优先级 | 动作 | 预期效果 |
|--------|------|---------|
| P0 | 新用户引导流程：注册后根据选择领域推荐模型 | 提升首次调用率20% |
| P0 | 一键示例功能：无需输入即可体验模型效果 | 降低体验门槛 |
| P1 | 实时在线客服：5分钟内响应使用问题 | 减少技术支持流失 |
| P1 | 首次调用失败自动诊断：常见错误自动提示修复 | 提升首次调用成功率 |
| P2 | 新用户邮件序列：使用技巧、案例分享 | 提升7日留存 |

---

### 转化节点2：免费用户 → 付费用户（Revenue Conversion）

**当前行业基准**：SaaS产品平均2-5%，优秀产品可达10%+

**Nexify目标**：≥ 8%

**付费触发场景**：

| 触发场景 | 用户心理 | 产品动作 |
|---------|---------|---------|
| API额度用完 | "这个工具对我有用，值得付费" | 额度预警 + 付费引导 |
| 需要Pro模型 | "免费模型不够用，需要更强的" | Pro模型预览 + 限时试用 |
| 团队协作需求 | "团队都需要用，需要共享账号" | 团队版功能预览 + 邀请奖励 |
| 私有化部署需求 | "数据不能上云，需要本地部署" | 企业版咨询入口 + 方案介绍 |

**产品动作**：

| 优先级 | 动作 | 预期效果 |
|--------|------|---------|
| P0 | 额度预警机制：用量达到80%时提示升级 | 提前触发付费意愿 |
| P0 | Pro模型限时试用：让用户体验付费价值 | 提升付费转化率 |
| P1 | 灵活的付费方案：按量付费/包月/包年选项 | 满足不同用户需求 |
| P1 | 团队版试用：小团队免费试用14天 | 降低团队决策门槛 |
| P2 | 付费用户专属内容：高级教程、专家分享 | 提升付费用户粘性 |

---

### 转化节点3：个人用户 → 企业客户（Enterprise Conversion）

**当前行业基准**：自然转化率<1%，主动销售转化率5-10%

**Nexify目标**：≥ 5%

**企业需求信号**：

| 信号 | 识别方式 | 跟进动作 |
|------|---------|---------|
| 多账号同公司注册 | 邮箱域名分析 | 主动联系，提供团队版试用 |
| API调用量激增 | 使用量监控 | 客户成功经理介入 |
| 下载私有化部署包 | 下载行为追踪 | 企业版方案推送 |
| 咨询定制开发 | 客服/邮件咨询 | 销售团队跟进 |

**产品动作**：

| 优先级 | 动作 | 预期效果 |
|--------|------|---------|
| P0 | 企业版功能入口：在控制台明显位置展示 | 提升企业版认知 |
| P0 | 自动识别企业用户：同域名多账号聚合 | 发现潜在企业客户 |
| P1 | 客户成功经理分配：高价值用户专人跟进 | 提升企业转化率 |
| P1 | 企业案例展示：同行业成功案例 | 建立信任 |
| P2 | 企业用户专属活动：线下沙龙、行业峰会 | 建立关系 |

## 用户流失预警信号

### 早期预警（可挽回）

| 信号 | 定义 | 自动干预 |
|------|------|---------|
| **注册后7天无调用** | 注册后7天内API调用次数为0 | 发送"需要帮助吗？"邮件，提供1对1支持 |
| **Demo体验后无后续** | 体验了Demo但没有下载/调用 | 推送相关模型案例，激发使用场景 |
| **API错误率上升** | 近7天错误率>10% | 自动诊断邮件，提供解决方案 |
| **文档跳出率高** | 文档页面停留<30秒 | 优化文档结构，增加视频教程 |

### 中期预警（需主动干预）

| 信号 | 定义 | 干预措施 |
|------|------|---------|
| **使用量下降50%** | 近30天用量环比上月下降>50% | 客户成功经理电话回访 |
| **未使用新功能** | 上线30天后未尝试新功能 | 功能介绍邮件/演示邀请 |
| **竞品调研行为** | 搜索竞品关键词后访问减少 | 差异化价值强调 |
| **支持工单增加** | 近30天工单数>5 | 技术负责人直接对接 |

### 晚期预警（流失风险高）

| 信号 | 定义 | 挽留措施 |
|------|------|---------|
| **连续30天无登录** | 30天内无任何活动 | 发送"我们想念你"邮件，提供回归优惠 |
| **取消订阅意向** | 访问取消订阅页面 | 弹出挽留问卷，提供折扣/延期 |
| **负面反馈** | 提交差评/投诉 | CEO/高管直接回复，优先解决问题 |
| **竞品迁移** | 导出数据量异常增加 | 紧急客户成功介入，了解原因 |
